Intelligence via Memory

Extrag is a high-performance, Agentic ETL-to-RAG engine designed for systems that need more than just search. It transforms static documents into a dynamic memory layer that evolves based on real-world usage and feedback loops.

Engine Architecture

Extrag is built as a high-performance orchestration layer that bridges raw filesystem data with agentic intelligence. The engine operates across four primary planes:

1. The Ingestion Plane (Delta-ETL)

The ingestion lifecycle begins with the FilesystemExtractor, which performs recursive directory traversal. To ensure maximum efficiency, every file is passed through our BLAKE3 Fingerprinting stage. By comparing content hashes against the SyncStateStore, the engine identifies exactly which documents require re-vectorization, effectively eliminating redundant LLM API calls for static data.

2. The Processing Plane (Rust-Native)

Documents designated for ingestion are streamed into the Processing Pipeline. Here, the RecursiveCharacterChunker performs context-aware splitting, maintaining the structural integrity of your data. These chunks are then vectorized in parallel using Tokio-based concurrency, utilizing the full potential of your system's CPU and networking to achieve high-throughput embedding generation.

3. The Retrieval Plane (MemRL)

The Retrieval Engine implements a Value-Encoded Search pattern. When a query is received, the engine performs a hybrid pass: it calculates the semantic similarity (spatial distance in the vector store) and fetches the historical utility ($Q$-score) for high-probability candidates. These distinct metrics are fused using Z-Score normalization, allowing the engine to surface "Trusted Knowledge" even when semantic matches are ambiguous.

4. The Feedback Plane (Reinforcement)

The engine's intelligence is self-evolving. Every match returned to a user or agent can be programmatically rewarded or penalized via the Feedback API. These delta-rewards are applied directly to the vector payloads in Qdrant, persistently updating the knowledge base's global utility map. This creates a long-term memory that "remembers" which document chunks were actually helpful in solving real-world tasks.

Technical Stack: Built entirely in Rust for memory safety and zero-cost abstractions, Extrag utilizes Axum for its REST interface, Qdrant for vector persistence, and Ollama as the primary embedding and inference backbone.

Delta-Aware Agentic ETL

Traditional RAG systems re-index entire datasets, leading to massive API costs. Extrag implements Delta-Vectorization to solve this.

Content Fingerprinting

Extrag hashes every file content using BLAKE3. If a file's hash matches the previous ingestion state in our SyncStateStore, it is skipped entirely, saving compute and embedding costs.

Semantic Chunking

Instead of fixed-window splits, our RecursiveCharacterChunker respects document structure (paragraphs, lists, code blocks) to preserve context. This reduces "broken context" errors in retrieval.

Pro Tip: Use the /v1/cache/clear endpoint to force a full re-sync if you update your embedding model or chunking strategy.

MemRL RAG & Hybrid Scoring

Retrieval in Extrag is Action-Aware. The engine doesn't just ask "What is similar?", it asks "What has been useful?"

Z-Score Normalization

Since semantic scores and utility rewards are in different ranges, Extrag normalizes both into Z-scores before fusion. This ensures that a few high rewards don't completely drown out semantic relevance.

The Utility Function

Rewards (feedback) are additive. A chunk's Q-score starts at 0 and grows or shrinks based on +1.0 or -1.0 increments from the system or human experts.

Hypothetical Document Embeddings (HyDE)

Extrag bridges the vocabulary gap by generating a 1-2 sentence "Pseudo-Answer" via the LLM before search. We find that embedding the answer yields much higher precision than embedding the question alone.

Powering Agentic Systems

Extrag is designed to be the Long-Term Memory (LMT) for autonomous agents. By integrating with the /v1/feedback endpoint, an agent can "learn" over time.

Self-Healing Knowledge

Unhelpful or outdated information naturally decays in rank over time as it receives negative feedback, ensuring the "sharpest" information stays at the top of the context window.

Tool-Augmented Retrieval

Agents can use Extrag as a Tool. If the agent finds a retrieved chunk contains a hallucination, it can programmatically penalize that chunk, automatically preventing future hallucinations from the same source.

Agent Integration: When building agents, automate feedback by penalizing results if the agent's internal "Verification Step" fails after processing the retrieved context.

API Reference

Interact with the engine via a high-performance REST API. All requests accept and return application/json payloads.

POST /v1/ingest

Initiates a delta-aware ingestion of a filesystem directory. Only new or modified files will be re-indexed.

curl -X POST http://localhost:8080/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{"path": "/data/docs"}'
const response = await fetch('/v1/ingest', {
  method: 'POST',
  body: JSON.stringify({ path: '/data/docs' })
});
import requests

res = requests.post(
    "http://localhost:8080/v1/ingest",
    json={"path": "/data/docs"}
)
POST /v1/retrieve

The primary retrieval endpoint. Use this to pull contextual chunks fused with their utility profiles.

curl -X POST http://localhost:8080/v1/retrieve \
  -H "Content-Type: application/json" \
  -d '{"query": "MemRL Logic", "top_k": 3}'
const res = await fetch('/v1/retrieve', {
  method: 'POST',
  body: JSON.stringify({ query: '...', top_k: 3 })
});
res = requests.post("...", json={"query": "..."})
POST /v1/feedback

Submit a reward or penalty to a specific piece of context to update its Q-score.

curl -X POST http://localhost:8080/v1/feedback \
  -d '{"document_id": "...", "reward": 1.0}'
fetch('/v1/feedback', {
  method: 'POST',
  body: JSON.stringify({ document_id: '...', reward: 1.0 })
});
requests.post("...", json={"reward": 1.0})
GET /v1/collections

Returns a list of all existing collections in the vector store.

curl http://localhost:8080/v1/collections
const data = await (await fetch('/v1/collections')).json();
collections = requests.get("...").json()
